# podman_apps

This directory contains configuration, deployment, and management scripts for containerized applications running via Podman with systemd quadlets

---

## Managing applications

Since all apps are managed by systemd, the same commands work for every app. Replace `<app>` with the app name (e.g. `vpn-downloader`)

**Deploy:**
```bash
systemctl --user enable --now <app>
```

**Stop:**
```bash
systemctl --user stop <app>
```

**Restart:**
```bash
systemctl --user restart <app>
```

**Status:**
```bash
systemctl --user status <app>
```

**Logs:**
```bash
journalctl --user -fu <app>
```

---

## Viewing running containers and pods

**List all pods:**
```bash
podman pod ps
```

**List all containers:**
```bash
podman ps
```

Each pod has a `stack` label that can be used to filter these commands to a specific app:
```bash
podman pod ps --filter label=stack=<app>
podman ps --filter label=stack=<app>
```

## First-time setup
```bash
# 1. Clone the repo and run the installer
./installApps.sh
```

`installApps.sh` will automatically check for missing secrets and prompt for input before deploying each app

Secrets can also be managed manually via:
```bash
# Create or overwrite secrets interactively
/mnt/podman/<app>/manageSecrets.sh
```

## Updating after a git pull
```bash
git pull
./installApps.sh
```

`installApps.sh` is idempotent — it can be run as many times as needed. It recreates all symlinks, reloads systemd, and restarts all apps to pick up any changes

## installApps.sh

This script installs all Podman-based applications by creating symlinks from the repo into their target locations, reloading systemd, and deploying all apps

It performs the following actions:

1. **Takes ownership of `/mnt/podman`** — creates `/mnt/podman` if it doesn't exist and ensures the current user owns it
2. **Enables linger** — runs `loginctl enable-linger` so rootless containers survive logout
3. **Container definitions** — symlinks all files in `apps/*/containers` into `/mnt/podman/<app>/containers/`
4. **Quadlets** — symlinks all files in `apps/*/quadlets` into `~/.config/containers/systemd/`
5. **Scripts** — symlinks all files in `apps/*/scripts` into `/mnt/podman/<app>/`
6. **Reloads systemd** — runs `systemctl --user daemon-reload` once after all symlinks are created
7. **Checks secrets** — runs `manageSecrets.sh --check` for any app that has one, prompting for input if secrets are missing
8. **Deploys all apps** — enables and starts each app via `systemctl --user enable --now`

## apps

This directory contains all application-specific files and folders

### `containers/`
Holds the Kubernetes-style Pod YAML that defines the application — images, environment variables, resource limits, volume mounts, and secrets. Symlinked into `/mnt/podman/<app>/containers/` by `installApps.sh`

### `quadlets/`
Contains quadlet unit files (`.kube`) that tell systemd how to manage the application. Systemd generates a `.service` unit from these automatically after `daemon-reload`. Symlinked into `~/.config/containers/systemd/` by `installApps.sh`

### `scripts/`
Contains management scripts for human use. Symlinked into `/mnt/podman/<app>/` by `installApps.sh`. Common scripts include:

- **`manageSecrets.sh`** — creates or overwrites the Kubernetes Secret manifest (`secrets.yaml`) used by the app. Run with `--check` to only create if missing, or with no flags to prompt for all values

---

## Application data

Runtime data for all applications lives under `/mnt/podman/<app>/.data/` on the dedicated podman disk mounted at `/mnt/podman`. This directory is created automatically by `DirectoryOrCreate` volume mounts in the Pod YAML — no manual setup required

---

## Secrets

Secrets are stored as Kubernetes Secret manifests (`secrets.yaml`) generated by `manageSecrets.sh`. These files contain sensitive credentials and are **gitignored** — they are never committed to the repo
